Help Defining Core Steps

A Project Manager has identified the core steps crucial to the success of their projects. For each step in the example below, they have outlined the process happening in each section, specified who will be involved, and how the tasks will transition from one step to the next within their workflow. The Project Manager's approach ensures they have a comprehensive understanding of the entire process.

Example

Task Start & Scope Definition​

Is the staring point in the workflow where all project related tasks are created.

  • The Project Manager creates tasks in this section based on the set project objectives and the established creative direction. Each task will contain detailed instructions for the work to be accomplished and include links to materials or other resources for the Creative Team to start work on the tasks. The Project Manager will @mention and assign the task to the Creative Lead for review.

  • The relevant Creative Leads review the task descriptions and materials to ensure the instructions provided are sufficient for the Creative Team to start working on the task.  

  • Sufficient Task Description
  • Insufficient Task Description
  • The Creative Lead writes a comment with an @mention to notify the Project Manager about who should be assigned the task for production.
  • The Project Manager assigns the task and sets the following Custom Fields:
    • Due Date: using a specified range for production time.
    • Creative Team: a tag for tracking tasks through the workflow for KPIs.
    • Assignee: assign to the Creative Team member to put the task on their Asana task list.
    • Status: set to 'production,' and Asana moves the task to the Production step.
  • The Creative Lead writes a comment @mentioning the Project Manager with what adjustments need to be made to the instructions and list any missing materials or resources your team needs to commence work on the task. Set the 'Status' field to ‘Insufficient’ and Asana moves the task to the Insufficient Information section.

 

Production​

The Creative Team is actively working on tasks.

  • After completing the work, the task's assignee will set the 'Status' field to 'Inspection.' This action notifies the relevant Creative Lead that the work is ready to be reviewed. Asana unassigns the task and moves it to the Inspection step.

 

Inspection​

The Creative Lead reviews the completed work, ensuring it meets internal quality standards.

  • Asana creates an 'Approval Request,' sets a due date, and assigns the approval request to the Creative Lead.
  • Approved
  • NOT Approved

The Creative Lead marks the request as 'Approved' and sets the 'Status' field to 'Validation.' This action notifies the Project Manager that the work is ready to be reviewed and Asana moves the task to the Validation section.

The Creative Lead marks the request as 'NOT Approved,' writes a comment explaining what adjustments need to be made to the work, and @mentions the assignee and Project Manager. The Creative Lead reassigns the task back to the original assignee and sets the 'Status' field to 'Production.' Asana moves the task back to the Production section.

 

Validation​

The Project Manager reviews the completed work, ensuring it meets internal quality standards, customer objectives, and the creative direction.

  • Asana creates an approval request with a due date assigned to the Project Manager.
  • Approved
  • NOT Approved

The Project Manager marks the request as 'Approved' and sets the 'Status' field to 'Customer Evaluation.' Asana creates a subtask with a due date assigned to the Project Manager. The Project Manager marks the subtask as complete when they have submitted the work to the customer and Asana moves the task to the 'Submitted for Customer Evaluation' step.

The Project Manager marks the request as 'NOT Approved,' writes a comment explaining what adjustments need to be made to the work, and @mentions the assignee and Creative Lead. The Project Manager sets the task's due date range, reassigns the task, and sets the 'Status' field to 'Production.' Asana moves the task to the Production step.

 

Submitted for Customer Evaluation​

The Project Manager has submitted the completed work to the customer for their review and feedback.

  • Asana assigns the task to the Project Manager and sets a specified due date range for the task.
  • Modifications Requested
  • No Modifications

The Project Manager changes the 'Status' field to 'Modifications,' after the customer has completed their evaluation and requested changes. Asana moves the task to the Modification step.

The Project Manager marks the task as complete, after the customer has completed their evaluation and no modications are requested. Asana changes the 'Status' field to 'Done,' and moves the task to the Done section.

 

Modifications Requested​

The Project Manager processes customer feedback within a set time period.

  • Asana sets a specified due date range and reassigns the task to the Project Manager.

  • The Project Manager updates the task's description with the customer's feedback. The updated description will contain detailed instructions for the modifications to be made to the work, along with any materials or other resources for the Creative Team.

  • The Project Manager sets the task's due date range, reassigns the task, and sets the 'Status' field to 'Production.' Asana moves the task to the Production step.

 

Done​

Is the endpoint of the workflow. All tasks in this section have been finalized and handed over to the customer.

  • Asana unassigns and moves all Project tasks marked complete to this section. This is the end of the project workflow.

 

Insufficient Information​

This section contains tasks that do NOT have sufficient instructions or materials for the Creative Team to commence work.

  • Asana moves tasks with the 'Status' field set to ‘Insufficient’ to this section, assigns the task to the Project Manager, and sets a specified due date.

  • The Project Manager reviews the instructions and/or missing resources, and obtains further information or materials from the customer for the Creative Team to begin work on the task. The Project Manager will @mention and assign the task to the Creative Lead for review. The Creative Lead completes the same task review steps outlined in the 'Task Start & Scope Definition' section.
